= Configure seu cluster ONTAP do sistema de armazenamento AFX
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Após a instalação do hardware AFX, você pode concluir a configuração do cluster ONTAP. Esse processo envolve fases de configuração relacionadas que devem ser executadas sequencialmente. Como parte disso, você pode opcionalmente adicionar os nós de computação de dados (DCNs) que executam o AI Data Engine (AIDE), permitindo que o cluster de storage AFX e o cluster DCN operem como um único sistema gerenciado pelo mesmo plano de controle ONTAP.



== Prepare-se para configurar um sistema de storage AFX

Antes de configurar seu cluster ONTAP, você deve revisar os requisitos principais do AFX e as considerações adicionais do AIDE para garantir que tenha todas as informações e recursos necessários para concluir o processo de configuração com sucesso.



=== Requisitos principais do AFX

Existem vários requisitos essenciais a serem considerados ao configurar seu cluster AFX.

.Instalação de hardware
Certifique-se de ter concluído a instalação de hardware do seu sistema AFX. Para mais informações, consulte link:../install-setup/install-setup-workflow.html["Instale seu sistema AFX"].

.Endereçamento IP básico
Você precisa da seguinte configuração de endereçamento IP:

* Endereço IP de gerenciamento de cluster
+
O endereço IP de gerenciamento do cluster é um endereço IPv4 exclusivo para a interface de gerenciamento do cluster usado pelo administrador do cluster para acessar o SVM do administrador e gerenciar o cluster.  Você pode obter esse endereço IP com o administrador responsável por atribuir endereços IP na sua organização.

* Máscara de sub-rede de rede
+
Durante a configuração do cluster, o ONTAP requer um conjunto de interfaces de rede apropriadas para sua configuração.  Você pode ajustar a recomendação se necessário.



.Configuração de rede adicional
Você também precisará das seguintes informações de rede relacionadas:

* Endereço IP do gateway de rede
* Nomes de domínio DNS
* Endereços IP do servidor de nomes DNS
* Endereços IP do servidor NTP
* Máscara de sub-rede




=== Considerações adicionais sobre o AIDE

Ao instalar e configurar o AFX pela primeira vez, você pode incluir opcionalmente NetApp AI Data Engine (AIDE). AIDE oferece suporte a dois modelos de implantação: software AIDE em servidores de terceiros ou nós de computação de dados (DCNs) fornecidos pela NetApp. Se você planeja configurar AIDE, conclua primeiro os pré-requisitos para o seu modelo de implantação:

* Para implantações de servidores de terceiros, conclua  https://docs.netapp.com/us-en/ai-data-engine/install-aide-server/aide-mde-server-hardware-requirements.html["instalação de hardware para AI Data Engine em servidores de terceiros"^] e, em seguida,  https://docs.netapp.com/us-en/ai-data-engine/install-setup/install-aide-as-software.html["instalar o software AIDE em servidores de terceiros"^].
* Para implantações de NetApp DCN, conclua  https://docs.netapp.com/us-en/ai-data-engine/install-dcn/aide-dcn-hardware-requirements.html["instalação de hardware para AI Data Engine em nós de computação de dados fornecidos pela NetApp"^].
* Obtenha os endereços e quaisquer configurações de rede relacionadas aos nós de computação.




== Executar configuração inicial do cluster

Você pode conectar seu dispositivo laptop ao cluster AFX e definir vários valores de configuração global.

.Sobre esta tarefa
Há quatro áreas do cluster AFX que devem ser configuradas inicialmente.  Os três primeiros são obrigatórios, enquanto o último é opcional.

.Passos
. Descubra sua rede de cluster.
+
.. Conecte seu laptop ao switch de gerenciamento e acesse os computadores e dispositivos da rede.
.. Abra o Explorador de Arquivos.
.. Selecione *Rede*; clique com o botão direito e selecione *Atualizar*.
.. Selecione o ícone ONTAP e aceite quaisquer certificados exibidos na tela.
+
A interface do usuário do Gerenciador do Sistema é exibida.



. Defina a senha do administrador.
+
Forneça e verifique a senha para o `admin` conta. Selecione *Continuar*.

. Configure os endereços IP para o cluster AFX e os nós do controlador.
+
Forneça endereços IP e máscaras de sub-rede.

. Se você estiver instalando AI Data Engine com seu sistema de storage AFX, forneça os endereços IP dos três nós de computação.
. Configure os serviços de rede, incluindo DNS e NIS. Selecione *Continue*.
+
Defina os detalhes dos seus servidores DNS e NTP.

. Opcionalmente, configure a criptografia.
+
Você pode definir os detalhes para criptografia de cluster. Selecione *Continuar*.

+

NOTE: Para obter informações sobre como criar um cluster para um ambiente que não seja Windows, consulte link:https://docs.netapp.com/us-en/ontap/software_setup/setup-cluster.html["Crie um cluster ONTAP e junte os nós."^].



.O que vem a seguir
Você será redirecionado para a página de login do Gerenciador de Sistema.  Execute os passos descritos em<<Configuração completa do cluster>> .



== Configuração completa do cluster

Após a configuração inicial ser realizada, você pode concluir a configuração do cluster ONTAP usando o Gerenciador do Sistema.

.Sobre esta tarefa
Há três áreas do cluster ONTAP do sistema AFX configuradas durante a instalação.  Complete todos os três, se possível, mas somente o primeiro é necessário.

.Antes de começar
Você precisa ter as seguintes informações:

* Detalhes de configuração da VLAN.
* Detalhes de configuração do NAS e/ou S3.


.Passos
. Sign in no Gerenciador do Sistema usando a conta de administrador fornecida durante a configuração inicial do cluster.  Observe a janela pop-up no canto superior direito com três opções de configuração.
. Selecione *VLAN e marcação* e selecione as opções de rede apropriadas para seu ambiente.
. Selecione *Serviços de rede* e configure os protocolos de acesso do cliente para o SVM de dados padrão.
. Selecione *Contêiner de dados* e crie um volume ou bucket S3.


.O que vem a seguir
* Se você incluiu AIDE como parte da configuração do cluster AFX, conclua as etapas adicionais de configuração do AIDE. A configuração de autenticação se aplica a todas as implementações do AIDE. O licenciamento é necessário para implantações NetApp DCN que usam recursos licenciados do AIDE; implantações de servidores de terceiros são focadas em metadados e não exigem uma licença do AIDE. Consulte  https://docs.netapp.com/us-en/ai-data-engine/install-setup/cluster-setup-afx.html["Configurar AI Data Engine"^] para mais informações.
* Você develink:../get-started/prepare-cluster-admin.html["Prepare-se para administrar o AFX"] antes de usar seu cluster AFX em um ambiente de produção.
